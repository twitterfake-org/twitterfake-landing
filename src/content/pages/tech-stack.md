# Stack Tecnológico de TwitterFake

## Backend - Spring Boot Ecosystem

### Framework Principal
- **Spring Boot** - Framework principal para desarrollo rápido y robusto
- **Spring Security** - Gestión de autenticación y autorización
- **Spring Data JPA** - Capa de persistencia con Hibernate

### Base de Datos
- **MySQL** - Sistema de gestión de base de datos relacional
- **JPA/Hibernate** - ORM para mapeo objeto-relacional

### Arquitectura
- **Arquitectura Hexagonal** - Separación clara de responsabilidades
- **Patrón Repository** - Abstracción de la capa de datos
- **Inyección de Dependencias** - Gestión automática con Spring IoC

## Frontend - Tecnologías Modernas

### Framework y Librerías
- **Angular** - Framework SPA para aplicaciones dinámicas
- **TypeScript** - Superset de JavaScript tipado
- **RxJS** - Programación reactiva y gestión de observables

### Styling y UI
- **TailwindCSS** - Framework CSS utility-first
- **PrimeNG** - Biblioteca de componentes UI para Angular
- **Responsive Design** - Adaptable a todos los dispositivos

## Seguridad y Autenticación

### Implementación de Seguridad
- **JWT (JSON Web Tokens)** - Autenticación sin estado
- **OAuth2** - Integración con proveedores externos (Google)
- **HTTPS** - Comunicación segura
- **CORS** - Control de acceso cross-origin

### Control de Acceso
- **Roles de Usuario** - guest, user, admin
- **Autorización basada en roles** - Permisos granulares
- **Validación de entrada** - Sanitización de datos

## Testing y Calidad

### Testing Framework
- **JUnit 5** - Framework de testing para Java
- **Mockito** - Framework de mocking para pruebas
- **Patrón AAA** - Arrange, Act, Assert

### Métricas de Calidad
- **Cobertura de código** - Pruebas unitarias exhaustivas
- **Análisis estático** - Detección de code smells
- **Testing de integración** - Validación end-to-end

## DevOps y Despliegue

### Plataformas de Despliegue
- **Firebase** - Hosting del frontend
- **Koyeb** - Despliegue del backend
- **GitHub Actions** - Pipeline CI/CD

### Automatización
- **CI/CD** - Integración y despliegue continuo
- **Docker** - Containerización para consistencia
- **Environment Management** - Configuración por entornos

## Herramientas de Desarrollo

### IDEs y Editores
- **IntelliJ IDEA** - IDE para desarrollo Java
- **Visual Studio Code** - Editor para frontend

### Control de Versiones
- **Git** - Sistema de control de versiones
- **GitHub** - Repositorio remoto y colaboración

### Gestión de Dependencias
- **Maven** - Gestión de dependencias Java
- **npm** - Gestión de paquetes Node.js

## ¿Por qué estas Tecnologías?

La selección del stack tecnológico se basó en:

1. **Escalabilidad** - Tecnologías probadas en entornos enterprise
2. **Mantenibilidad** - Código limpio y arquitectura modular
3. **Seguridad** - Frameworks con seguridad built-in
4. **Productividad** - Herramientas que aceleran el desarrollo
5. **Comunidad** - Soporte activo y documentación extensa

Este stack demuestra el dominio de tecnologías modernas y mejores prácticas en el desarrollo de aplicaciones web escalables.
